# P-020: Seed database with development fixture data

## Objective

Provide a repeatable way to populate the database with a small,
coherent dataset for local development and testing.

## Scope

- Add a seed script inside `apps/web`
- Insert:
  - 1 user
  - 1 installation
  - 1 tariff plan with at least 2 versions (date-ranged)
  - A small set of energy readings spanning multiple days
- Data must exercise:
  - tariff version boundaries
  - billing logic paths

## Out of Scope

- No UI work
- No external data ingestion
- No large datasets

## Expected Implementation

- Seed script located at:
  - `apps/web/src/db/seed.ts` (or similar)
- Add npm script:
  - `pnpm db:seed` (or equivalent)

## Constraints

- Data must be:
  - deterministic
  - readable
  - small enough to inspect manually

## Dependencies

- Existing schema from P-018
- Domain logic from P-019

## Acceptance Criteria

- Running the seed command inserts a valid dataset
- Data spans tariff version windows
- Data can be queried by existing domain logic
- No errors during seed execution

## Tests

- Optional: basic test verifying seed inserts expected entities

## Notes

- This data will be used by P-021 (first billing read path)
