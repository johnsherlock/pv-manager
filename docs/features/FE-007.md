# FE-007: Auth and Onboarding Foundation

## Goal

Turn the rewrite from a seeded local-user app into a real account-based product
that can admit beta users safely, authenticate them through Google, and block
normal app access until each approved user has connected a valid provider
account.

This feature establishes the minimum account and onboarding foundation for beta:

- approval-gated sign-in
- terms/privacy acceptance as part of account creation
- a required provider-credentials gate before app entry
- secure capture of provider credentials per installation
- a clear handoff to Settings for later optional setup

## Status

`Todo`

## Feature Completion Criteria

- Users can sign in with Google and be created with app-managed role/status
  metadata rather than seed-only hacks
- Unapproved users are blocked cleanly on a waitlist / invite-only screen
- Admin can approve users and notify them without a tokenized invite-link flow
- Approved users are blocked on a provider-setup gate until valid provider
  credentials have been verified server-side
- Provider credentials are captured server-side and linked to the persisted
  provider connection rather than deployment-level env vars
- Later optional setup is explicitly deferred to Settings rather than built as a
  multi-step onboarding wizard
- Auth, approval, provider-gate, and impersonation edge cases are defined
  clearly enough for later implementation and verification

## Deferred From This Feature

- Gifted pro access and premium entitlement management
- Broader admin approval tooling beyond what auth/provider gating needs
- Public marketing, welcome, and demo pages beyond the minimal beta-entry path
- Optional tariff, finance, solar, and location setup flows beyond the required
  provider step

## Notes

- This feature is intentionally about making the app usable by real beta users,
  not about broad self-serve growth or premium packaging.
- Public access-request UX and demo mode are grouped separately under the public
  and unauthenticated experience feature.
