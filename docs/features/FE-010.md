# FE-010: Admin Operations Console

## Goal

Create an operator-only area for running the beta safely: managing users,
reviewing approvals and entitlements, monitoring provider/job health, and using
support-safe operational tools without mixing those controls into normal user
settings.

## Status

`Todo`

## Feature Completion Criteria

- Admin-only surfaces are clearly separated from end-user settings
- Beta approvals, user management, and entitlement controls have a defined home
- Operational diagnostics and job visibility have a product-level surface rather
  than living only in ad hoc scripts or hosting dashboards
- Support-safe tooling respects privacy and avoids exposing plaintext secrets

## Included Stories

Stories currently grouped here:

1. [`P-014`](../backlog.md) Design account entitlements and gifted pro access
2. [`U-018`](../backlog.md) Design admin approvals and gifted access screens
3. [`Q-006`](../backlog.md) Create provider diagnostics and reconciliation analysis tool

## Deferred From This Feature

- Provider or organization tenant management beyond the beta admin model
- Full Data Health product reporting, which remains grouped under `FE-002`
- Production-grade cron/job controls beyond what beta operators immediately need

## Notes

- This feature gives the newly discussed admin area a real roadmap home even
  though several detailed implementation stories still need to be created.
- Job execution/logging infrastructure is grouped with beta delivery and
  deployment rather than the admin UI itself.
