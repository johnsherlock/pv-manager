## Q-009: Rewrite Workflow Scope

The `v2` branch is currently treated as the rewrite line for this repository.

### GitHub Actions

- `.github/workflows/build.yml` runs on pushes to `v2`, pull requests targeting `v2`, and manual dispatch.
- The build job runs from `apps/web`.
- The job installs dependencies with `npm ci`, runs `npm run test`, and then runs `npm run build`.
- `.github/workflows/pull-request-lint.yml` only validates PR titles for pull requests that target `v2`.

### Legacy App Placement

- The original application and its legacy tooling now live in `V1/` at the repository root.
- This keeps the rewrite branch root focused on the new app in `apps/web` while preserving the earlier implementation for reference and migration support.
