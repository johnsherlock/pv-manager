# FE-008: User Settings and Tariff Management

## Goal

Give signed-in users the screens they need to maintain the data that powers the
product: tariff versions, contract reminders, notification-related settings, and
other user-editable setup that should not require support intervention.

## Status

`Todo`

## Feature Completion Criteria

- Users can review the currently active tariff and historical tariff versions
- Users can edit tariff and contract details with clear recalculation impact
- Reminder and tariff-validity states are surfaced clearly
- Settings/edit flows feel safe, understandable, and consistent with the rest of
  the app

## Included Stories

Stories in intended execution order:

1. [`U-027`](../backlog.md) Build polished clickable prototype for the Tariffs overview screen
2. [`U-028`](../backlog.md) Build polished clickable prototype for tariff editing and contract reminders
3. [`U-013`](../backlog.md) Add tariff-validity and contract reminders

## Deferred From This Feature

- Admin-only controls
- Entitlements and premium access controls
- Data Health and trust-reporting surfaces

## Notes

- This feature covers user-facing maintenance flows, not operator workflows.
- Notification preferences may land here later, but they do not yet have their
  own backlog stories.
