# FE-004: Live Weather and Solar Context

## Goal

Extend the lower `Next` section of the Live screen so it gives users useful,
location-aware context about the next few hours without turning the product into
a separate weather app.

This feature should:

- show the next 12 hours of forecast data in a horizontally scrollable strip
- show the next 5 days of forecast data in a compact daily view
- show sunrise, sunset, solar noon, daylight remaining, and current sun position
- gate the feature cleanly when location is missing
- support privacy-aware onboarding with exact or approximate location capture
- stay explicitly limited to zero-cost API options for the initial delivery

## Status

## Feature Completion Criteria

- Onboarding supports both exact and approximate location modes
- The product stores canonical latitude/longitude plus a user-selected precision mode
- The Live screen can render useful daylight and sun-context information without
  making roof-orientation, panel-tilt, or shading claims
- The 12-hour and 5-day forecast modules are part of the Live screen, not a
  separate weather page
- Documented provider choices remain zero-cost for the initial implementation path
- Provider constraints, caching expectations, and fallback behavior are documented
- Missing-location and partial-failure states remain coherent and useful on Live

## Provider Direction

Initial documented shortlist:

- Open-Meteo for weather forecast and daily sun-event data
- Open-Meteo Geocoding and/or Geoapify free tier for geocoding evaluation

Explicitly out of scope for the initial feature:

- Google Maps as a required dependency, because it requires billing
- Public Nominatim as the primary app dependency, because its published usage
  policy is too restrictive for product use at scale

## Deferred From This Feature

- Roof orientation, tilt, or shading modeling
- Panel-output prediction from weather alone
- Paid APIs
- Severe-weather alerts, notifications, or automation
- Historical weather backfill beyond what the Live screen needs
- AI recommendations tied to weather

## Notes

- This feature is a Live-screen enhancement, not a new weather product area
- Approximate location support is part of the initial scope, not a later privacy enhancement
- Solar context should remain sky-relative and honest about what the product does
  not know
- The goal is to improve interpretation of Live data, not to compete with a
  dedicated weather app
