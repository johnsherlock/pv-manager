# FE-012: Beta Delivery and Deployment

## Goal

Make the rewrite shippable and operable as a real beta product by tightening the
delivery workflow, deployment path, observability expectations, and release
checks around `apps/web`.

## Status

`Todo`

## Feature Completion Criteria

- The team has a documented beta release workflow
- PR and branch expectations are explicit and enforced consistently
- Preview and shipping deployment workflows exist for the rewrite app
- Operational logs and job execution strategy are coherent enough for beta
  support
- Privacy, deletion, and core correctness checks have a defined verification
  path as the product moves beyond local-only use

## Included Stories

Stories in intended execution order:

1. [`P-016`](../backlog.md) Set up GitHub execution access
2. [`Q-001`](../backlog.md) Define validation strategy
3. [`Q-004`](../backlog.md) Define beta release workflow
4. [`Q-010`](../backlog.md) Define and enforce PR rules for the rewrite branch
5. [`Q-011`](../backlog.md) Add rewrite preview build workflow
6. [`Q-012`](../backlog.md) Add rewrite deployment workflow for the main shipping branch
7. [`P-006`](../backlog.md) Centralize job execution and logs
8. [`Q-005`](../backlog.md) Define privacy and deletion verification

## Deferred From This Feature

- Broad operational analytics or BI work beyond beta support needs
- Full admin-console UX for job management and support actions
- Product-level outage/trust reporting beyond deployment and release readiness

## Notes

- This feature gathers the delivery and operations work that was previously
  scattered across the backlog without a clear feature home.
