# FE-001: Live Single-User Slice

## Goal

Deliver one end-to-end working Live experience in `apps/web` for a single seeded
local user, using:

- seeded rewrite DB data for installation and tariff context
- the temporary V1 minute-data backend for current-day minute-level telemetry

The purpose of this feature is to prove the rewrite can move from prototype to
working product without waiting on onboarding, auth, or multi-user concerns.

## Status

`In Progress`

## Feature Completion Criteria

- Local Postgres setup is documented and working
- Seeded installation and tariff data load successfully
- The rewrite can fetch and normalize current-day minute data server-side
- The Live route renders from real data rather than prototype-only values
- Current-day health states are surfaced from real heuristics
- Core verification exists for the seeded billing slice and tariff/billing regression path
