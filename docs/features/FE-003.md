# FE-003: Historical Day View

## Goal

Deliver the first dedicated Historical Day experience in `apps/web` so users can
review a past day as a completed daily story rather than as a slightly modified
Live screen.

This feature should:

- create a separate historical day route/page for past dates
- reuse extracted single-day analysis components instead of duplicating one large screen
- remove realtime-only and future-looking modules from historical dates
- use neutral day wording instead of `today` language
- make day-to-day browsing fast on desktop and touch devices through caching,
  prefetch, and swipe navigation

## Status

`Todo`

## Feature Completion Criteria

- Past dates render through a distinct Historical Day page/route rather than the
  Live screen shell
- Shared single-day analysis modules are extracted from `LiveScreen` and reused
  across Live and Historical Day
- Historical Day shows day totals, day value, day trend, solar coverage, and a
  historical notes/interpretation section
- Historical Day hides realtime metric cards, realtime interpretation, and
  next-few-hours / solar-context modules
- Historical labels use neutral day wording and no longer incorrectly imply the
  selected day is `today`
- The header clock continues ticking independently of whether the user is on
  Live or Historical Day
- Adjacent-day caching and prefetch make date-to-date browsing responsive
- Mobile/tablet users can swipe left/right between days without breaking chart
  interaction or normal scrolling
- Navigating forward from a historical day back to the current day returns the
  user to Live

## Notes

- `P-027` was already in use in the backlog for outage/anomaly notifications, so
  the enabling refactor story for this feature is tracked as `P-029`
- This feature is intentionally focused on single-day history first, not
  multi-day/range reporting
- Adjacent-day prefetch is not just a performance nicety here; it is part of the
  interaction quality required to make button and swipe navigation feel native
- Historical interpretation should stay grounded in selected-day data and trust
  state, without overreaching into unsupported year-over-year or ranking claims
