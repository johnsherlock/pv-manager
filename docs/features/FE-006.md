# FE-006: Range History Screen

## Goal

Build the Range History screen in `apps/web`, giving users a period-level view of
their energy use, solar performance, and financial outcomes across any chosen date
range. Range History answers "how am I doing over time?" — leading with financial
outcome and solar contribution before breaking down into energy charts.

This feature begins with a foundational data fix (P-039) that adds rate-band import
breakdown to daily summaries, making accurate time-of-use billing possible from
summary data without needing to store raw interval readings.

## Status

`Done`

## Feature Completion Criteria

- Rate-band import breakdown (day/peak/night) is stored in `daily_summaries` and
  used by the billing domain to produce accurate per-band cost calculations
- The range-summary API exposes band-level import kWh in the day series
- A Range History page exists at a clear route in `apps/web`
- Period selector supports named presets and a custom date range path
- Summary header shows correct banded financial totals, completeness, and
  tariff-change indicators
- Energy trend chart and per-day stacked bar chart are implemented (without the
  redundant third bar from v1)
- Financial cost histogram and period cost breakdown chart are implemented using
  accurate banded billing
- Solar coverage and export ratio charts are implemented
- Payback progress is shown for financed installations
- All states are handled: missing data, partial days, tariff-change span,
  no-tariff/savings-locked, loading, and hard error
- Mobile and desktop layouts follow the direction from U-040
- The preset pill row is replaced by an advanced tabbed range picker popover
  with Custom range / Weeks / Months / Years tabs, flanking chevrons that step
  by the current selection increment, and a preset strip
- Zooming any ECharts chart syncs the active range across all charts and the
  picker, and the chevrons step by the zoomed window width

## Deferred From This Feature

  column in P-039 but the classification logic is deferred until the
  `free_import_rule_json` tariff field schema is formally defined
- Rate-band breakdown in the no-solar baseline — the no-solar counterfactual
  continues to use the day rate as a known simplification; accurate time-of-use
  no-solar modeling requires assumptions about when in the day consumption would
  have occurred without solar, which is deferred

- Best/worst day callouts and percentile insights
- Export of range data
