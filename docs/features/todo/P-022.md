# P-022: Build first server-side estimated billing read path

## Objective

Create the first vertical slice of the rewrite application by loading persisted
daily summaries and tariff data, running billing domain logic, and returning a
structured estimated billing result.

## Scope

- Implement a server-side route or page in `apps/web`
- Load:
  - installation
  - tariff data (including versioning)
  - daily summaries for a requested period
- Invoke existing domain logic from `src/domain/billing.ts`, adapting it only as
  needed for the summary-first model
- Return or render an estimated billing summary

## Out of Scope

- No provider API calls
- No UI polish
- No authentication
- No supplier bill reconciliation

## Expected Implementation

- Add a route or server-side handler under:
  - `apps/web/src/app/...`
- Fetch data via the existing DB layer
- Call the billing domain service
- Return structured output suitable for app rendering or inspection

## Constraints

- Keep logic server-side only
- Reuse existing billing logic where practical instead of re-implementing calculations
- Prefer a shape that can later support both page rendering and API-style consumption

## Dependencies

- Seed data from P-021
- Domain logic from P-019

## Acceptance Criteria

- Route or page loads successfully
- Billing logic executes without errors against persisted daily summaries
- Output includes:
  - estimated total cost
  - estimated savings
  - key breakdown values
- Works against seeded development data

## Tests

- Add integration-style test or simple verification test
- Ensure the returned output shape is stable and verifiable

## Notes

- This is the first summary-backed vertical slice: DB -> domain -> app output
- Prioritize correctness and inspectability over UI polish
