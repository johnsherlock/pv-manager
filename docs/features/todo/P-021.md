# P-021: Build first server-side billing read path

## Objective

Create the first vertical slice of the rewrite application by loading stored data,
running billing domain logic, and returning a structured billing result.

## Scope

- Implement a server-side route or page in `apps/web`
- Load:
  - installation
  - tariff data (including versioning)
  - energy readings
- Invoke existing domain logic (from `src/domain/billing.ts`)
- Return or render a billing summary

## Out of Scope

- No UI polish
- No authentication
- No multi-user concerns
- No external API calls

## Expected Implementation

- Add a route or server-side handler under:
  - `apps/web/src/app/...`
- Fetch data via existing DB layer
- Call billing domain service
- Return structured result (JSON or simple rendered output)

## Constraints

- Must use existing domain logic (do not reimplement calculations)
- Keep logic server-side only
- Avoid introducing new abstractions unless necessary

## Dependencies

- Seed data from P-020
- Domain logic from P-019

## Acceptance Criteria

- Route/page loads successfully
- Billing logic executes without errors
- Output includes:
  - total cost
  - savings
  - key breakdown values
- Works against seeded data

## Tests

- Add integration-style test or simple verification test
- Ensure expected output structure is returned

## Notes

- This is the first full vertical slice (DB → domain → app)
- Prioritise correctness over UI
