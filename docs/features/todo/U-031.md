# U-031: Reframe the Live screen around realtime state, live value, and solar context

## Objective

Create a second-pass polished clickable prototype for the Live screen that builds on
`U-023` and turns the current baseline into a more differentiated product surface.

This story should keep the Live experience realtime-first, but make it more useful
and memorable by combining:
- current live system state
- same-day financial value so far
- short-horizon solar and daylight context

The result should feel meaningfully stronger than a standard inverter dashboard or a
theme variation of the first prototype.

## Scope

- Revise the Live screen prototype on:
  - desktop
  - mobile
- Preserve the existing top-level Live purpose:
  - answer what the system is doing right now
  - show whether the data is fresh and trustworthy
- Expand the Live screen to include three clear information layers:
  - `Now`
  - `Today so far`
  - `Next`
- Reuse the strongest useful ideas from V1 where they improve understanding:
  - date / today context
  - scale controls such as minute / half-hour / hour
  - chart mode controls such as line / cumulative
  - legend toggles for dense chart series
  - live energy usage snapshot treatment
- Introduce a secondary financial interpretation layer for users with tariff data:
  - import cost so far
  - export value so far
  - savings so far
  - net bill impact so far
- Introduce a site-aware solar context layer for users with coordinates:
  - sunrise / sunset
  - daylight remaining
  - current sun altitude / azimuth
  - solar noon or remaining productive window
  - weather-aligned near-term solar outlook
- Include the most important prototype states:
  - healthy / default
  - stale data
  - warning / suspicious data
  - disconnected provider
  - provider-only setup where tariff and/or site inputs are missing

## Out of Scope

- No production implementation in `apps/web`
- No Data Health screen redesign beyond CTA/deep-link treatment
- No full weather-forecast product design beyond what supports the Live screen
- No roof-orientation modeling or shading analysis
- No claim about sun position relative to the house or array unless orientation data
  becomes part of a separate future story
- No reopening of the broader signed-in IA from `U-020`

## Expected Output

- One revised polished clickable desktop prototype for Live
- One revised polished clickable mobile prototype for Live
- A clear information hierarchy covering:
  - realtime energy state
  - live or same-day financial value
  - solar/daylight/forecast context
- State variants showing how the screen behaves when:
  - tariff data is missing
  - installation capacity is missing
  - coordinates are missing
  - data is stale, suspicious, or disconnected
- A short rationale note explaining:
  - how the revised screen is more distinctive than the first prototype
  - how the screen balances telemetry, value, and context
  - how provider-only users still get a coherent experience

## Constraints

- The Live screen must remain realtime-first, not become a finance dashboard
- Trust and freshness must remain visible near the top of the screen
- Financial interpretation must be secondary to current live state
- Missing tariff data must remove or replace only the value modules, not the energy modules
- Missing installation capacity must still hide efficiency gracefully
- Missing coordinates must hide solar-context modules gracefully
- Coordinates alone may support daylight and sun-position context, but not roof-relative
  or array-relative claims
- The revised screen should feel structurally different from the first prototype, not
  like a dark/light or cosmetic variation
- Avoid flattening the page into many equal-weight cards; use stronger hierarchy and grouping
- The primary chart must remain interpretable even with multiple series enabled

## Dependencies

- `U-023`
- `U-020`
- `docs/ui/mid-fi-layouts.md`
- `docs/ui/data-contracts.md`
- current `apps/web` Live prototype as reference
- V1 Live dashboard as reference material only

## Acceptance Criteria

- A revised clickable Live prototype exists for desktop and mobile
- The prototype clearly presents three layers of understanding:
  - what is happening now
  - what today's behavior has meant so far
  - what is likely to happen over the next few hours
- The Live screen shows:
  - current generation
  - current consumption
  - current import
  - current export
  - freshness / trust state
  - solar coverage or grid reliance
  - a primary live chart with usable controls
  - current-day totals
- When tariff data exists, the Live screen also shows:
  - import cost so far
  - export value so far
  - savings or net bill impact so far
- When tariff data is missing, those value modules become a prompt card rather than fake values
- When coordinates exist, the Live screen shows daylight / sun-context information that
  improves interpretation without overclaiming installation-specific accuracy
- When coordinates are missing, the screen remains coherent without those modules
- The revised screen feels more distinctive and product-specific than the first prototype

## Review / Validation

- Review whether the screen answers more than one useful question without becoming cluttered
- Review whether the financial layer adds product value without displacing realtime clarity
- Review whether the solar-context layer feels genuinely helpful rather than decorative
- Review whether the chart controls improve readability when multiple lines overlap
- Review whether a provider-only user still gets a useful Live experience
- Review whether the revised desktop and mobile versions preserve the same hierarchy

## Notes

- This story is a refinement follow-up to `U-023`, not a replacement for it
- The goal is to capture the stronger direction explicitly so implementation does not
  default back to the flatter first-pass prototype
- A future story can explore optional roof orientation / pitch inputs if we later want
  installation-relative sun explanations or expected-production modeling
