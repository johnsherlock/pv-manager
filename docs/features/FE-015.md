# FE-015: Range History Financial Interpretation

## Goal

Turn Range History into a trustworthy, data-rich financial interpretation
surface that explains what solar created, what happened to the energy bill, and
whether the system is paying for itself.

This feature should replace the current ambiguous "Solar savings" framing with
a clearer financial model that separates:

- energy value
- bill impact
- investment payoff

It should lead with a simple hero narrative for the selected period, then allow
users to dig into richer supporting detail without making the screen feel like a
black box.

## Status

`Todo`

## Feature Completion Criteria

- Range History financial metrics have explicit, user-facing definitions rather
  than overloaded or misleading labels
- The primary hero story for the selected period is clear and consistent:
  - solar created value
  - value came from self-consumption and export
  - repayments cost a known amount
  - net solar position is visible
- The screen separates:
  - energy value metrics
  - bill impact metrics
  - investment payoff metrics
- Actual tariff-priced metrics use the real tariff data available for the
  period rather than silently flattening everything to a day-rate assumption
- The product distinguishes between:
  - actual measured energy totals
  - tariff-priced estimates
  - counterfactual "without solar" estimates
- "Self-consumed solar value" is defined separately from export credit and from
  broader bill-delta metrics
- Range History can surface richer supporting financial data without forcing all
  metrics into the top summary row
- Immersion diversion can be interpreted as a trade-off against export rather
  than disappearing into raw energy totals
- Automated coverage includes worked financial examples that protect the
  wording/model from drifting out of sync with the calculations

## Notes

- `FE-006` established the first production Range History screen and its core
  charts. This feature is about making the financial interpretation layer worthy
  of being the product's main long-range value surface.
- `FE-014` introduces the first investment-aware payoff modules. This feature
  should build on that work, not replace it, by making the surrounding value
  model clearer and easier to trust.
- The key product distinction in this feature is:
  - `self-consumed solar value` is not the same thing as export credit
  - `total solar value` is not the same thing as bill reduction
  - `net solar position` is not the same thing as energy bill
- Another key distinction is:
  - actual tariff-priced outcomes should use the real tariff structure we have
  - counterfactual no-solar outcomes may remain estimated, but only with an
    explicit method and explicit labeling
- The first implementation does not need to settle the final UI packaging
  pattern. It is acceptable to define the metric model and surface hierarchy
  first, then decide later whether the detailed sections are shown as stacked
  cards, tabs, collapsible groups, or a denser dashboard.
