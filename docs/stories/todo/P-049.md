# P-049: Build investment recovery and payoff insight surfaces

## Objective

Implement the first finance-driven insight surfaces so users can see recovered
value, repayment coverage, and a simple payoff estimate in the signed-in app,
using the finance settings model from `P-047` and the UX rules from `U-051`.

## Scope

- Extend signed-in loaders/API payloads to expose the finance insight data needed
  by the chosen screens
- Implement the first recovery/payback modules defined by `U-051`
- Surface:
  - recovered value from available recorded history
  - estimated total recovery since investment date when needed
  - selected-period savings vs repayment coverage
  - simple payoff estimate based on historical savings behavior
- Ensure missing-finance state degrades to clear setup prompts rather than fake zeroes
- Ensure incomplete-history state explains confidence limits honestly
- Reuse existing financial calculations where possible rather than creating a
  separate finance math path
- Add automated coverage for:
  - upfront-only
  - repayments-only
  - mixed upfront-plus-repayments
  - partial-history scenarios
  - selected-period repayment coverage
  - payoff-estimate labeling

## Out of Scope

- Multi-investment-event support
- Advanced forecasting or weather-informed payoff prediction
- Full Data Health or outage reporting UI

## Acceptance Criteria

- Signed-in users with finance setup can see the first recovery/payback insights
- Users without finance setup see a prompt state instead of misleading values
- Partial-history users see recorded vs estimated recovery distinguished clearly
- Selected-period repayment coverage reflects the configured finance mode accurately
- Payoff estimate is visibly labeled as approximate guidance
- Tests cover the main happy paths and partial-history cases

## Dependencies

- `P-047`
- `P-048`
- `U-051`
- Existing range/overview financial surfaces
