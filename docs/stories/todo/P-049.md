# P-049: Build multi-investment recovery and payoff insight surfaces

## Objective

Implement the first finance insight surfaces unlocked by recorded system
additions so users can see:

- recovered value so far
- repayment coverage for the selected period
- a simple, clearly caveated payoff outlook

The first pass should follow `U-051`: whole-system focused, centered in Range
History, and built on the app's existing financial model rather than a new
recovery math path.

## Scope

- Extend signed-in loaders/API payloads to expose the recovery/payoff insight
  data needed by the chosen screen(s), with Range History as the primary home
- Implement the first recovery/payback modules defined by `U-051`
- Surface:
  - `Recovered so far` using cumulative net savings from recorded history
  - `Repayment coverage for selected period` using selected-period savings vs
    repayments due in that same selected period
  - `Approximate payoff outlook` when the available history is strong enough to
    support a useful estimate
- Show whole-system totals only in the first pass; do not implement
  per-investment recovery attribution in this story
- Ensure missing-investment state degrades to clear setup prompts rather than fake zeroes
- Ensure incomplete-history state explains confidence limits honestly and
  distinguishes:
  - recorded recovery from available stored history
  - estimated total recovery since investment date when shown
- Omit payoff or total-recovery estimates when the estimate would be too weak
  or misleading
- Reuse existing financial calculations where possible rather than creating a
  separate finance math path
- Add automated coverage for:
  - one-record installations
  - multi-record installations
  - upfront-only records
  - repayment-backed records
  - mixed portfolios containing both
  - partial-history scenarios
  - selected-period repayment coverage for month/year/custom range views
  - estimate-omission behavior when history is insufficient
  - payoff-estimate labeling

## Out of Scope

- Advanced forecasting or weather-informed payoff prediction
- Full Data Health or outage reporting UI
- Detailed per-investment recovery attribution
- A separate lifetime-recovery screen or dashboard beyond the first chosen
  surface(s)

## Acceptance Criteria

- Signed-in users with investment records can see the first recovery/payback insights
- Users without investment records see a prompt state instead of misleading values
- `Recovered so far` is implemented using cumulative net savings from recorded history
- Partial-history users see recorded recovery vs estimated total recovery
  distinguished clearly when both are shown
- Selected-period repayment coverage reflects the configured investment records
  accurately for month, year, and custom range views
- The first pass is explicitly whole-system focused and does not include
  per-investment recovery attribution
- Payoff estimate is visibly labeled as approximate guidance
- Weak or misleading estimates are omitted rather than forced into the UI
- Tests cover the main happy paths and partial-history cases

## Dependencies

- `P-051`
- `P-052`
- `U-051`
- Existing Range History financial surfaces
