# U-015: Design sign-in, waitlist, and gated-entry UX

## Objective

Define the user experience after sign-in so each beta-access state is obvious:
awaiting approval, approved but blocked on provider setup, approved and ready,
or suspended.

## Scope

- Define the signed-out entry messaging and sign-in CTA
- Define the `awaiting_approval` screen:
  - invite-only / waitlist message
  - sign-out only action
- Define the suspended-user response
- Define the post-approval entry behavior
- Define how the provider-setup gate is introduced after approval
- Define the "sign in with the same Google account" guidance in the UX
- Define how terms/privacy acceptance appears in the sign-in / first-use flow

## Out of Scope

- No public demo mode design
- No beta request form design
- No multi-step onboarding wizard for later optional setup
- No detailed admin-console UX beyond what auth gating needs

## Dependencies

- `P-013`

## Acceptance Criteria

- The post-sign-in outcomes are documented for unapproved, approved,
  suspended, and admin users
- The waitlist screen is intentionally minimal and includes only the actions
  needed for this beta phase
- The provider-setup gate is positioned as a required access step, not an
  optional wizard
- Copy expectations make the MyEnergi requirement visible early and clearly
- Terms/privacy touchpoints are clear enough for implementation

## Notes

- Keep this flow simple: sign in, get approved, connect provider, then use the
  app
- Optional tariff, finance, solar, and location setup should not leak back into
  this story
