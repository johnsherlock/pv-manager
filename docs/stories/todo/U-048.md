# U-048: Build the required provider-setup gate and Settings handoff

## Objective

Create the focused provider-setup experience that approved users must complete
before they can enter the app, while keeping all later optional setup work in
Settings rather than in a broader onboarding wizard.

## Scope

- Gate all normal user-app routes until a valid provider connection exists
- Present provider setup as a focused first destination after approval
- Provider field shape:
  - provider selector / provider-shaped entry
  - credentials form for the selected provider
- For the first version, only MyEnergi is supported
- Include a "How do I find my MyEnergi credentials?" help affordance
- On success, redirect the user into the normal app
- After entry, route later optional setup to Settings rather than continuing
  through a multi-step onboarding flow

## Out of Scope

- No tariff, finance, solar, or location setup UI
- No public demo mode
- No multi-provider support beyond a future-ready form shape
- No customer-editable provider history or advanced reconnect management

## Dependencies

- `U-015`
- `P-038`

## Acceptance Criteria

- Approved users without a valid provider connection are redirected to the
  provider-setup gate instead of the normal app
- The gate is focused on provider connection only and does not become a broader
  onboarding wizard
- The first version makes MyEnergi the only supported provider while preserving
  a provider-shaped UX structure
- Help content exists explaining how users find their MyEnergi credentials
- After successful provider validation, the user can enter the app and later
  optional setup lives under Settings

## Notes

- Structurally this can live inside Settings, but in the user journey it should
  behave like a full-page access gate until complete
