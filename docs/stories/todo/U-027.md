# U-027: Build polished clickable prototype for the Tariffs overview screen

## Objective

Turn the existing tariff-management direction into a polished clickable Tariffs
overview that makes current truth, historical versions, reminder state, and the
active weekly schedule easy to understand at a glance.

## Scope

- Current tariff summary
- Compact weekly schedule summary / preview
- Version timeline / history list
- Reminder / expiry state
- Main actions into create/edit flows
- Desktop and mobile layouts
- Empty, stale, and setup-incomplete states

## Out of Scope

- No production implementation in `apps/web`
- No tariff editor field-level validation behavior
- No full Settings shell design beyond what the Tariffs section needs

## Dependencies

- `U-020`
- `U-013`
- `P-044`

## Design Reference

- [`docs/ui/settings.md`](../../ui/settings.md) — Settings shell context,
  Tariffs section definition, and how the Tariffs overview fits within the
  Settings navigation model (produced by U-049)

## Acceptance Criteria

- A polished clickable prototype exists for the Tariffs overview on desktop and
  mobile
- The screen makes `what is true now` distinct from historical tariff changes
- The current tariff includes a legible summary of its weekly 30-minute schedule
  without forcing the overview to become an editor
- Reminder / expiry state is visible without overwhelming the main tariff
  information
- Empty and stale states still preserve clear next actions

## Notes

- This should feel like a trustworthy product surface, not an internal tariff
  admin table
