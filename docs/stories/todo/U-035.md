# U-035: Add first-pass Historical Day notes and interpretation

## Objective

Replace the live/future-looking lower section with a historical notes and
interpretation block that helps the user understand the selected day without
pulling later comparative features into the first implementation slice.

## Scope

- Add a concise historical notes/story block driven by selected-day data
- Cover interpretation themes such as:
  - generation strength
  - grid reliance
  - export vs self-consumption mix
  - trust/completeness warnings
  - tariff-availability-aware messaging

## Out of Scope

- Same-day-last-year comparisons
- Percentile ranking
- Best-day this month/year callouts
- Multi-day comparison logic

## Constraints

- The section must feel historical, not realtime or predictive
- Notes should be driven by selected-day data already available to the page
- Financial interpretation must degrade cleanly when tariff data is missing
- Trust/completeness language must stay calm and explicit

## Dependencies

- `U-034`
- `P-025`

## Acceptance Criteria

- The bottom section feels historical rather than realtime
- Notes are driven by selected-day data and trust state
- Tariff-missing states omit unsupported financial claims
- Partial-day states remain understandable and clearly caveated

## Tests

- Complete-day state
- Partial-day state
- No-tariff state
- Trust-warning state

## Notes

- Future stories can extend this section using `daily_summaries`
- This story intentionally stops short of rankings, percentile comparisons, and
  prior-year comparisons
- There's a UI bug on mobile that's preventing any touch controls to work. It was most likely introduced in the last story (U-034) when we introduced swipe controls on the UI (these also don't work).
