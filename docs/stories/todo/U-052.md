# U-052: Define Location and tariff-reminder settings UX

## Objective

Define the remaining global non-investment Settings domains so Location and
tariff-reminder preferences can replace their placeholder states with clear
real-world edit flows that stay tightly within the beta scope.

## Scope

- Define the `/settings/location` screen
- Define the `/settings/notifications` screen as tariff-reminder preferences
  rather than a broad notifications center
- Define the completion rules for:
  - Location
  - tariff-reminder preferences
- Define how users can clear/remove previously entered setup values in each
  domain and how that affects completion state
- Location:
  - reuse the existing exact/approximate model
  - keep precision only as strong as weather/daylight features require
  - define edit, save, geocode-success, geocode-failure, and unset states
- Tariff reminders:
  - define the initial user-manageable reminder preferences scope
  - keep wording aligned with tariff validity, not supplier contracts
- Define the Settings-home card treatment and value-unlock copy for each section

## Out of Scope

- Outage/anomaly notification delivery
- Rich notification-channel preferences
- Admin/operator settings

## Acceptance Criteria

- Location and tariff-reminder flows are documented clearly enough to implement
  without reopening their basic scope
- Location explicitly reuses the current exact/approximate approach
- Notification wording uses tariff-reminder language, not contract language
- Settings-home completion rules and summary states are defined for both domains
- Clear/remove behavior is documented where it is supported, including
  returning a domain to incomplete when appropriate
- Desktop and mobile behavior are documented
