# U-054: Define Range History financial metric model and narrative

## Objective

Define a clearer financial interpretation model for Range History so the page
can explain, for any selected period:

- what financial value solar created
- what happened to the energy bill
- what solar financing cost
- whether the system was ahead or behind overall

This story should replace ambiguous "solar savings" language with explicit
metric definitions and a clearer narrative hierarchy.

## Scope

- Define the primary hero narrative for the selected period, centered on:
  - solar created value
  - how that value split between self-consumption and export
  - repayments due in the period
  - net solar position for the period
- Define the first-pass financial metric groups for Range History:
  - Energy value
  - Bill impact
  - Investment payoff
- Define the intended meaning, formula basis, and plain-English explanation for
  each first-pass metric, including:
  - self-consumed solar value
  - export credit
  - total solar value
  - import bill
  - net energy bill
  - import bill without solar
  - bill reduction from solar
  - repayments
  - net solar position
  - recovered so far
  - payoff outlook
- Define tariff-accuracy rules for the metric set, including:
  - which metrics must use the actual tariff structure available for the period
  - which metrics may use an estimated or counterfactual method
  - how estimated or simplified tariff treatment must be labeled in the UI
- Define which metrics are:
  - measured totals
  - tariff-priced estimates
  - counterfactual estimates
- Define the relationship between the hero metrics and the deeper supporting
  detail so the screen can be data-rich without becoming incoherent
- Define wording rules that avoid double-counting confusion, especially around:
  - export credit vs total solar value
  - solar value vs bill reduction
  - repayments vs energy bill
- Define which existing Range History financial metrics should be retired,
  relabeled, or preserved only as secondary detail

## Out of Scope

- Final visual design treatment for the full page
- Immersion trade-off metrics beyond identifying where they belong conceptually
- Supplier-bill reconciliation or tax/accounting treatment
- New forecasting logic beyond the already agreed payoff guidance

## Acceptance Criteria

- Range History has a documented hero financial story for the selected period
- The first-pass metric catalogue is defined with explicit formulas or formula
  bases and plain-English descriptions
- The story explicitly defines which financial metrics must be priced using the
  actual tariff data available, and which are allowed to remain estimated or
  counterfactual
- The product clearly distinguishes energy value, bill impact, and investment
  payoff rather than blending them under one overloaded savings number
- The intended meaning of `self-consumed solar value`, `total solar value`,
  `bill reduction`, and `net solar position` is unambiguous
- The story identifies which current labels or calculations are misleading and
  need to be replaced or demoted
- The resulting spec is clear enough to implement without reopening the core
  financial terminology
