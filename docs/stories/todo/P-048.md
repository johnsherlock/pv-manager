# P-048: Build Finance settings screen and setup-completion flow

## Objective

Implement the real Finance settings screen in `apps/web`, replacing the current
placeholder with a simple finance form, saved summary state, and setup-completion
integration that unlocks downstream finance features.

This story is the production follow-up to `U-050` and should establish the
quality bar for the remaining non-tariff Settings domains.

## Scope

- Build the `/settings/finance` route/screen
- Implement the finance fields and validation UX for:
  - approximate investment date
  - upfront payment
  - monthly repayment amount including interest
  - repayment duration in months
- Save finance details through the server-side settings path introduced in `P-047`
- Show saved summary state that explains the chosen finance profile clearly
- Show derived summary values such as:
  - total investment amount
  - remaining repayment months
  - outstanding scheduled balance
- Update the Settings home Finance card so it reflects:
  - incomplete / actionable
  - complete
  - unlocked value messaging
- Handle empty, editing, save-success, and hard-error states
- Ensure mobile and desktop layouts remain coherent within the Settings shell

## Out of Scope

- Investment-recovery charts and projections outside the Finance settings page
- Multiple investment profiles
- Advanced inline calculators beyond derived total amount

## Acceptance Criteria

- Users can open a real Finance settings screen from Settings
- Users can enter a valid finance profile without being forced through a separate mode selector
- Save/update flow works and reloads the saved finance profile correctly
- The Finance card on `/settings` reflects completion state accurately
- The saved summary makes the user's finance setup understandable without
  reopening the form
- Desktop and mobile flows are usable and consistent with the Settings shell
