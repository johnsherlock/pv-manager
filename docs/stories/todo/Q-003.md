# Q-003: Add tariff transition tests

## Objective

Add explicit regression coverage for date-ranged tariff changes so billing outputs
remain correct when a requested period crosses from one tariff version into another.

## Scope

- Add tests that span at least one tariff version boundary
- Assert that tariff selection is resolved correctly per date
- Verify the resulting billing outputs remain correct across the boundary

## Out of Scope

- No UI work
- No provider-backed minute-data validation
- No contract-reminder UX

## Dependencies

- `F-006`
- `F-007`
- `P-019`
- `P-022`

## Acceptance Criteria

- At least one regression test spans a tariff transition period
- The test asserts the correct tariff version is applied on each side of the boundary
- The resulting totals and savings are stable and verifiable

## Notes

- This is a high-risk edge case and should stay isolated and obvious in the test suite

### Status: Planned
