# P-050: Build Location and tariff-reminder settings flows

## Objective

Implement the remaining global non-investment Settings screens so Location and
tariff-reminder preferences become real, user-editable parts of Settings
instead of placeholders.

## Scope

- Build the `/settings/location` route/screen and save flow
- Build the `/settings/notifications` route/screen for tariff-reminder preferences
- Location:
  - reuse the existing exact/approximate location capture model
  - support editing, saving, and re-geocoding from Settings
  - surface clear error treatment when geocoding fails
- Tariff reminders:
  - implement the first user-manageable reminder preferences
  - keep language and behavior aligned with tariff validity reminders only
- Update the Settings home cards and setup progress so these domains reflect
  real completion state instead of placeholders
- Support clearing/removing saved setup values where appropriate and return the
  affected domain to incomplete when minimum data no longer exists

## Out of Scope

- Outage/anomaly notification delivery
- Multi-channel notification management
- Weather or solar-context redesign beyond the location edit path

## Acceptance Criteria

- Users can open real Location and tariff-reminder screens
- Users can edit Location from Settings using the existing location model
- Users can clear/remove saved setup values where the UX allows it, and setup
  completion updates accordingly
- Geocode failure states are recoverable and clear
- Tariff-reminder preferences are editable and framed around tariff validity
- Settings home no longer shows these domains as generic placeholders
