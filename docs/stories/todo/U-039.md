# U-039: Build the Live weather, sunrise/sunset, and sun-position modules

## Objective

Define the implementation scope for wiring the lower Live modules so the `Next`
section can show forecast, sun events, and current sun position while preserving
the existing Live hierarchy and capability gating.

## Scope

- Wire the lower Live modules for:
  - 12-hour forecast rail
  - 5-day forecast cards
  - sunrise and sunset
  - solar noon
  - daylight remaining
  - current sun altitude and azimuth
- Preserve current Live hierarchy and trust treatment
- Keep setup prompts coherent when location is missing
- Keep solar-context claims honest and non-roof-relative
- Support horizontal scrolling for the 12-hour forecast module

## Out of Scope

- No paid API integration
- No roof orientation, tilt, or shading features
- No severe-weather notifications or automation
- No historical weather surfaces outside Live

## Expected Implementation

- Update the Live screen to replace the current placeholders in the `Next`
  section with production modules backed by the normalized server slice
- Preserve current capability gating and fallback prompts when location data or
  forecast data is unavailable
- Keep desktop and mobile behavior aligned with the documented content model

## Constraints

- The Live screen must remain realtime-first
- Weather and solar context must not visually overpower current telemetry
- Approximate-location states must be labeled clearly
- Implementation must follow the provider and contract decisions already captured
  in `P-033` through `P-035`

## Dependencies

- `U-038`
- `P-035`
- `U-031`

## Acceptance Criteria

- Implementation scope is specific enough for later coding without reopening API decisions
- Required lower Live modules are enumerated clearly
- Missing-location, approximate-location, and partial-failure behavior is defined
- Horizontal-scroll expectations for the 12-hour forecast are documented
- The story preserves the Live screen’s existing hierarchy and tone

## Tests / Verification

- Review the implementation scope against `LiveScreen` placeholders so follow-on
  work is concrete rather than exploratory

## Notes

- This is the production implementation story for the Live modules, but this doc
  itself is still requirements-only
- The feature should enhance interpretation of upcoming solar opportunity rather
  than promise generation forecasting accuracy it does not yet have
