# P-051: Extend data model for system additions and linked investment records

## Objective

Implement the persistence and domain changes needed to replace the flat
installation-level finance profile with a list of dated system-addition
records attached to one installation.

## Scope

- Add a first-class model for system-addition records
- Ensure one installation can have multiple dated records
- Persist the first-pass fields per record:
  - label / description
  - date
  - capacity added (`kWp`) when relevant
  - upfront payment
  - monthly repayment amount
  - repayment duration in months
- Derive per-record totals:
  - total investment amount
  - remaining repayment months
  - outstanding scheduled balance
- Expose normalized settings payloads for listing and editing records
- Remove or ignore the superseded flat finance shape rather than migrating it
  forward for beta users
- Add tests for:
  - multiple records on one installation
  - valid/invalid record combinations
  - derived totals
  - delete/remove behavior where required

## Out of Scope

- Full lender/supplier agreement modeling
- Refinancing-specific timeline logic
- Historical ROI calculations in product UI

## Acceptance Criteria

- One installation can store more than one dated system-addition record
- Server-side validation rejects invalid combinations
- Derived totals are deterministic per record
- Existing flat finance setup data does not block the new model and is not
  required to migrate forward for beta
- Settings loaders can distinguish empty vs configured multi-record state
- Test coverage exists for single-record and multi-record scenarios

## Dependencies

- `U-053`
- `P-020`
- Existing finance/settings persistence work
