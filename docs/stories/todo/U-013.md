# U-013: Add tariff-validity and contract reminders

## Objective

Surface reminders and warnings when tariff validity or contract review dates
have passed so users can keep the financial model trustworthy without having to
audit tariff history manually.

## Scope

- Define and surface expired-validity warnings
- Define and surface review-due reminder states
- Clarify the difference between tariff validity and contract reminder metadata
- Ensure reminder states appear coherently in Tariffs overview and related edit
  flows
- Define the recommended correction/update actions

## Out of Scope

- No outage or anomaly notifications
- No email reminder delivery system
- No admin reminder tooling

## Dependencies

- `U-005`
- `F-009`

## Acceptance Criteria

- Reminder and warning states are documented clearly enough for production
  implementation
- Users can tell whether a tariff is calculation-stale, contract-review-stale,
  or both
- Reminder states guide the user toward the right corrective action

## Notes

- This story is about product behavior and UX clarity; delivery channels for
  reminders can be layered on later
