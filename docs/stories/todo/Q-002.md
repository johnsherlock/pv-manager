# Q-002: Add billing parity tests

## Objective

Protect the billing logic by asserting that calculated outputs match the expected
results in the golden fixture set, especially now that the rewrite has a seeded DB
path and a first summary-backed billing slice.

## Scope

- Add tests that run the billing domain logic against `F-007` fixtures
- Assert expected values for:
  - import cost
  - fixed charges
  - export credit
  - net cost
  - savings
- Focus on correctness, not broad UI coverage

## Out of Scope

- No UI tests
- No live provider minute-data validation
- No end-to-end browser coverage

## Dependencies

- `F-007`
- `P-019`
- `P-022`

## Acceptance Criteria

- Fixture-backed billing parity tests exist
- Failures clearly identify which expected outputs drifted
- The tests are suitable for CI use

## Notes

- This is one of the main correctness gates before a wider beta path
