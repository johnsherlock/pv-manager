# U-037: Mobile layout improvements for Live screen

## Objective

Apply the same mobile layout improvements from U-036 to `LiveScreen.tsx` so the Live
screen is usable and readable on a 412×915 mobile viewport.

## Background

U-036 fixed the Historical Day screen. The Live screen has the same structural problems:
the control bar wraps to multiple rows on mobile, there is verbose descriptive text above
the chart, and the chart components (shared with Historical Day via `DayAnalysis.tsx`) were
already fixed in U-036 (chart heights, legend grid, eye icon alignment). This story covers
the Live-screen-specific chrome only.

Note: `DayAnalysis.tsx` chart heights, series legend layout, and chart card description
text were already made responsive in U-036 — those changes apply to the Live screen too.
This story focuses only on the Live-screen-specific layout in `LiveScreen.tsx`.

## Scope

- **Control bar** (`LiveScreen.tsx`): The Live screen has a more complex control bar than
  Historical Day — it includes the data-freshness badge, date/nav controls, live clock,
  uptime badge, and possibly a capability bar. Collapse to a single compact row on mobile,
  hiding secondary items (clock, uptime badge) behind `sm:` breakpoint, consistent with
  the Historical Day treatment.
- **Page hero / section headings**: Hide description paragraphs on mobile (`hidden sm:block`)
  in the Live-specific sections (e.g. "Current power, freshness..." subtitle).
- **Capability bar**: Assess whether it adds value on mobile or can be hidden/collapsed.

## Out of Scope

- Any changes to `DayAnalysis.tsx` (already done in U-036)
- Historical Day screen (done in U-036)
- New navigation patterns or information architecture

## Acceptance Criteria

- On a 412×915 mobile viewport the Live screen control bar occupies a single row
- The energy trend chart is visible above the fold (or with minimal scroll) on mobile
- No content is clipped or overlapping at 412px width
- Desktop layout (≥1024px) is unchanged
- All existing Live screen functionality works on mobile

## Notes

- `LiveScreen.tsx` is the only file expected to change
- The Live screen has more modules than Historical Day (realtime metric cards, capability
  bar, solar context) — assess each for mobile visibility vs hiding
- Aim for consistency with the Historical Day mobile treatment
