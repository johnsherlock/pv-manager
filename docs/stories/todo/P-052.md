# P-052: Build system additions and linked investment settings flows

## Objective

Implement the real Settings flow for managing dated system additions /
records, replacing the current flat finance-only form with a list and add/edit
experience that can represent original installs and later expansions.

## Scope

- Build the Settings route/screen for listing system-addition records
- Implement add/edit/delete flow for one record
- Implement the first-pass fields:
  - label / description
  - date
  - capacity added (`kWp`) when relevant
  - upfront payment
  - monthly repayment amount including interest
  - repayment duration in months
- Show saved summary state per record
- Show derived per-record summary values such as:
  - total investment amount
  - remaining repayment months
  - outstanding scheduled balance
- Update the Settings home/setup-completion treatment so this domain reflects
  empty, partial, and configured multi-record states
- Keep the UX understandable when only one record exists and when several exist

## Out of Scope

- Advanced bulk editing or drag-reordering
- Detailed lender management
- Downstream ROI/payback modules outside the setup/edit flow

## Acceptance Criteria

- Users can create more than one dated system-addition record
- Users can edit and review saved records clearly
- Derived totals are shown coherently per record
- Empty and multi-record states remain understandable
- Setup completion reflects the presence of meaningful configured records
- Mobile and desktop flows remain usable
