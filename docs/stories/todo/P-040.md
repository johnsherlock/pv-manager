# P-040: Build admin approval workflow and approval email

## Objective

Allow the seeded admin user to review registered users, approve users who are
awaiting access, and send the simple approval email that tells them to return to
the app and sign in again with the same Google account.

## Scope

- Admin-only user list showing at least:
  - name/email
  - role
  - status
  - created date
- Approve action for users in `awaiting_approval`
- Persist approval metadata:
  - `approved_at`
  - `approved_by`
- Send a plain approval email linking back to `www.solartracker.app`
- Include the exact-account reminder in the approval email
- Support "view app as user" for debugging/support with a persistent in-app
  banner during impersonation

## Out of Scope

- No full admin dashboard
- No invite-token generation
- No broader user-edit or suspension-management UI
- No audit-log persistence beyond normal application logs

## Dependencies

- `P-013`

## Acceptance Criteria

- Seeded admin can access the admin user-management surface
- Admin can approve a waiting user and the user's status changes to `approved`
- Approval records who approved the user and when
- Approval triggers a plain email telling the user to return to
  `www.solartracker.app` and sign in with the same Google account
- Admin can enter and exit "view app as user" mode with a persistent visual
  indicator while impersonation is active

## Notes

- This is intentionally the minimal admin slice needed to make the beta flow
  operable
- Detailed audit logging is deferred even though impersonation should still be
  treated carefully in the UX
