# U-045: Advanced range picker and chart zoom sync

## Objective

Replace the preset pill row on the Range History screen with an advanced range
picker popover. The picker exposes a tabbed calendar (Custom range / Weeks /
Months / Years), a set of presets, and `<` `>` chevrons that step the range
forward or backward by the same increment as the current selection. In addition,
zooming any ECharts chart on the page syncs the active range to the zoomed
window across all charts and the picker.

Reference screenshots are in `docs/ui/assets/range-picker/`.

---

## 1. Range picker — trigger and header bar

The current preset pill row is **removed**. In its place the sticky period bar
shows:

```
[<]   [Apr 2026]  [>]
```

- The label in the centre is a button that opens the picker popover.
- Label format varies by the active selection mode:

| Mode | Label example |
|------|--------------|
| Custom range (n days) | `10 Apr – 20 Apr 2025` |
| Weeks | `6 – 12 Apr 2026` |
| Months | `Apr 2026` |
| Years | `2026` |
| All | `All` |

- `<` steps backward by one unit (days, week, month, year).
- `>` steps forward by one unit.
- Both chevrons are **always rendered** but **disabled** (non-interactive,
  reduced opacity) when:
  - the selection would go beyond the earliest available data date (backward),
  - the selection would go beyond today (forward), or
  - the active mode is **All**.

---

## 2. Range picker — popover structure

The popover opens below the label button, anchored to the period bar. It is
dismissed by clicking outside or pressing Escape.

### 2a. Tab bar

```
[ Custom range ]  [ Weeks ]  [ Months ]  [ Years ]
```

Active tab has a filled green pill style (matching the reference screenshots).

### 2b. Custom range tab

- Calendar displays one month at a time in a **Su Mo Tu We Th Fr Sa** grid
  (Sunday-start weeks).
- Internal `<` `>` arrows step one month at a time within the calendar view.
- A year input/spinner at the top of the popover (`< 2025 >`) jumps the
  calendar year.
- Click interaction: first click sets the start date, second click sets the
  end date and closes the popover; clicking start again resets.
- The selected range is highlighted across the calendar cells between start and
  end.
- **Preset strip** at the bottom of this tab:

| Preset | Behaviour |
|--------|-----------|
| Today | Custom range: today–today |
| 7 days | Custom: today−6 → today |
| 30 days | Custom: today−29 → today |
| 90 days | Custom: today−89 → today |
| 12 months | Custom: today−364 → today |
| All | Sets mode to All; chevrons disabled |
| This week | Switches to Weeks tab; current Sun–Sat week selected |
| This month | Switches to Months tab; current month selected |
| This year | Switches to Years tab; current year selected |

### 2c. Weeks tab

- Same Su–Sa day-grid calendar as Custom range.
- Clicking any cell selects the entire Sun–Sat week that cell belongs to
  (cells outside the current month that fall in the same week are included).
- Selected week is highlighted as a contiguous block.
- Month navigation with `< >` arrows inside the calendar.
- Year spinner at the top.

### 2d. Months tab

- Displays a 4×3 grid of abbreviated month names (Jan … Dec) for the current
  year.
- Clicking a month selects that full calendar month (1st through last day).
- Year navigation with `< >` arrows at the top of the calendar section.

### 2e. Years tab

- Displays a 3×4 grid of years for the current decade (e.g. 2020–2029).
- The decade range is shown in the header: `< 2020 – 2029 >`.
- `<` / `>` step the visible decade backward/forward.
- Clicking a year selects Jan 1 – Dec 31 of that year.

## 3. Chevron step logic

When the user presses the outer `<` or `>` chevrons:

| Active mode | Step unit |
|-------------|-----------|
| Custom range of N days | Shift window by N days |
| Weeks | +/− 7 days |
| Months | +/− 1 calendar month |
| Years | +/− 1 year |
| All | No-op (chevrons disabled) |

A step that would push the window before the earliest available date or after
today is clamped and the chevron is disabled.

The earliest available date is derived from the minimum `local_date` in
`daily_summaries` for the installation. This can be returned as part of the
existing range API response (`meta.earliestDate`) rather than a separate
query.

---

## 4. "All" mode

- Fetches from `meta.earliestDate` (earliest available summary) through today.
- The server already loads a 365-day window; **All** requires extending the
  server-side fetch to the full available history.
- The period bar shows the label `All` with both chevrons disabled.
- The "All" preset strip button in the Custom range tab triggers this mode.

---

## 5. ECharts zoom sync

All ECharts instances on the Range History page (§4 Energy trend and §5
Per-day breakdown, plus §6–§9 when implemented) share a single active date
range managed by the screen's state.

When the user zooms using the ECharts dataZoom component (slider drag or
brush-select inside the chart):

- The zoomed window is translated back to a `from`/`to` date pair.
- All other charts on the page update to the same `from`/`to` window
  (i.e. all charts are controlled via shared `dataZoom` state, not per-chart).
- The picker label updates to show the zoomed range as a custom range (from–to
  dates).
- The chevron step size is set to the width of the zoomed window in days.
- Sync happens **on dataZoom change** (real-time during slider drag if the
  ECharts event fires continuously; on release if not).

Implementation note: use a shared `onDataZoom` handler wired to all chart
instances via `echarts-for-react`'s `onEvents` prop. The handler reads
`startValue`/`endValue` (category index or date string depending on axis type)
and converts them to `from`/`to` date strings, which are then lifted into the
shared range state.

---

## 6. Data loading changes

| Change | Notes |
|--------|-------|
| `meta.earliestDate` added to `RangeSummaryMeta` | Minimum `local_date` from the installation's daily summaries |
| Server load window extended for "All" mode | Requires a query param (e.g. `?mode=all`) or a second fetch when All is selected; the default 365-day window is unchanged for other modes |

## Acceptance Criteria

- [ ] Preset pills are removed from the sticky nav bar
- [ ] A range label + flanking `< >` chevrons appear in their place
- [ ] Clicking the label opens the range picker popover
- [ ] Custom range tab: calendar shows Su–Sa weeks; two-click date range
      selection works; selected range is highlighted
- [ ] Weeks tab: clicking any date selects the full Sun–Sat week
- [ ] Months tab: clicking a month name selects the whole calendar month
- [ ] Years tab: clicking a year selects Jan 1 – Dec 31
- [ ] Preset strip in Custom range tab: all presets set the correct date window;
      This week/month/year switch to the appropriate tab
- [ ] Label format matches the table in §1 for each mode
- [ ] Chevron step sizes match §3; chevrons disabled correctly at boundaries
      and in All mode
- [ ] All mode: loads full history; label shows "All"; chevrons disabled
- [ ] Zooming any chart syncs all other charts and the picker label
- [ ] After a chart zoom, chevrons step by the zoomed window width in days
- [ ] Picker closes on outside click or Escape
- [ ] Mobile: popover is usable on small screens (full-width or near-full-width)

## Out of scope

- Animation / transition between tabs
- Keyboard navigation within the calendar grid (can be a follow-up)
- Saving the last-used range across page loads
