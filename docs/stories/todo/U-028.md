# U-028: Build polished clickable prototype for schedule-based tariff editing and reminders

## Objective

Create a polished clickable prototype for the tariff create/edit flow so users
can correct or add tariff versions safely using a weekly 30-minute schedule,
while understanding validity windows, contract metadata, and recalculation
impact.

## Scope

- Create and edit modes
- Price-period definition
- Weekly 30-minute schedule editor
- Validity window and contract-detail section
- Schedule completeness / validation conflict states
- Recalculation impact messaging
- Reminder-related states where tariff validity or contract review is stale

## Out of Scope

- No production implementation in `apps/web`
- No final server-side validation rules
- No unrelated settings sections

## Dependencies

- `U-020`
- `U-013`
- `P-044`

## Design Reference

- [`docs/ui/settings.md`](../../ui/settings.md) — Settings shell and Tariffs
  section context; the editor flows are accessed from within
  `/settings/tariffs` (produced by U-049)

## Acceptance Criteria

- A polished clickable prototype exists for create/edit tariff flows
- The prototype centers on a weekly half-hour schedule editor rather than a
  small set of named time-window fields
- The prototype makes tariff validity distinct from contract reminder metadata
- Conflict and validation states are understandable and confidence-building
- Save-success messaging explains historical recalculation impact clearly

## Notes

- This should feel corrective and safe, not technical or back-office
- The target interaction is closer to the MyEnergi tariff editor than to a
  conventional start/end-time form
