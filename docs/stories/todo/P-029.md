# P-029: Extract shared single-day analysis components from LiveScreen

## Objective

Refactor the current monolithic Live screen into reusable single-day analysis
components so the historical day feature can reuse the same charts and summary
modules without duplicating one large page. This story is an enabling refactor
and should preserve current Live behavior while improving composition.

## Scope

- Extract reusable single-day presentation components from
  `apps/web/app/live/LiveScreen.tsx`
- Separate realtime-only sections from day-analysis sections
- Extract shared modules for:
  - day chart
  - day value panel
  - day totals panel
  - solar coverage panel
  - shared date header/navigation controls where useful
- Introduce copy/configuration props so shared modules can render live or
  historical wording
- Decouple the ticking clock from live auto-refresh logic

## Out of Scope

- No visual redesign
- No adjacent-day caching or prefetch
- No new historical notes logic
- No route changes by themselves

## Constraints

- Preserve current Live behavior while refactoring
- Avoid turning one component into a larger mode-switching component
- Realtime refresh remains Live-only
- Shared components must support neutral copy without hard-coding `today`

## Dependencies

- `U-030`

## Acceptance Criteria

- `LiveScreen` becomes a composition shell rather than one large page component
- Shared day-analysis components can be reused by Live and Historical Day
- Shared components support neutral labels suitable for historical rendering
- The header clock continues ticking independently of historical/live mode
- Live auto-refresh behavior remains intact and scoped to Live

## Tests

- Existing Live behavior remains intact after extraction
- The clock continues ticking while viewing historical dates
- Live auto-refresh still works for the current day
- Extracted components render correctly with alternate copy props

## Notes

- This story enables Historical Day reuse without committing to a single
  heavily conditional page
- It is intentionally refactor-first and user-invisible except for the clock
  behavior fix
