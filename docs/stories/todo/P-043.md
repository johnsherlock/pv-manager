# P-043: Build tariff version editor and contract reminder flows

## Objective

Implement the tariff create/edit experience in `apps/web`, including validity
window handling, contract metadata, overlap/conflict messaging, and honest
recalculation-impact guidance.

## Scope

- Create tariff version flow
- Edit tariff version flow
- Core tariff fields and rates
- Validity window handling
- Contract metadata fields
- Overlap / validation conflict behavior
- Success messaging that explains recalculation impact
- Reminder-related UI hooks that reflect stale validity or review dates

## Out of Scope

- No advanced bulk tariff import
- No admin tariff tooling
- No unrelated settings sections

## Dependencies

- `U-028`
- `P-042`

## Design Reference

- [`docs/ui/settings.md`](../../ui/settings.md) — Settings route structure
  and Tariffs section definition; editor routes should be nested under
  `/settings/tariffs` (e.g. `/settings/tariffs/new`, `/settings/tariffs/:id/edit`)
  (produced by U-049)

## Acceptance Criteria

- Users can create and edit tariff versions from the Tariffs section
- Validation prevents unsafe overlap or clearly explains conflicts
- Contract dates remain distinct from tariff validity in the UI and behavior
- Save-success messaging explains whether historical calculations may change
- Reminder-related states connect clearly to the corrective edit actions

## Notes

- This should feel like a trustworthy correction flow, not a raw data-entry
  form
