# P-054: Rework Range History financial calculations and presentation model

## Objective

Implement the clarified Range History financial model so the page presents a
trustworthy breakdown of solar value, bill impact, and investment payoff
instead of relying on the current ambiguous "Solar savings" KPI.

## Scope

- Rework the Range History financial summary layer to follow `U-054`
- Replace or relabel misleading KPI concepts so the selected period can surface:
  - total solar value
  - self-consumed solar value
  - export credit
  - net energy bill
  - import bill without solar
  - bill reduction from solar
  - repayments in the selected period
  - net solar position
- Ensure the hero financial story is visible without forcing all supporting
  metrics into the top summary row
- Separate calculations and labels so the UI does not imply:
  - export credit is extra on top of a total that already includes it
  - repayments are part of the energy bill
  - bill reduction and solar value are the same metric
- Ensure actual tariff-priced period outcomes use the real tariff information
  available for the covered days rather than collapsing everything to a simple
  day-rate assumption
- Where a metric still depends on a no-solar or avoided-cost counterfactual,
  implement the explicit estimation method defined in `U-054` and label the
  result accordingly
- Reuse existing persisted energy and tariff data where possible, but add or
  adjust range-side calculations when the current model does not match `U-054`
- Preserve explicit caveats where figures are estimated or counterfactual
- Update automated coverage for the revised metric model and wording-sensitive
  relationships

## Out of Scope

- Immersion trade-off comparison modules beyond any prerequisite data plumbing
- Supplier bill import/reconciliation
- New forecasting beyond the existing payoff guidance rules
- Final experimental UI packaging such as tabs vs stacked cards unless required
  to keep the screen understandable

## Acceptance Criteria

- Range History no longer relies on the current overloaded `Solar savings`
  concept as the primary headline financial KPI
- Hero and supporting financial metrics match the `U-054` model and are not
  internally contradictory
- Actual bill/import/export-related metrics are priced from the applicable
  tariff structure available for the selected period wherever the stored data
  supports that precision
- Any remaining simplified or counterfactual metric is clearly labeled as such
  rather than reading as a precise actual-bill figure
- The UI makes clear which values are:
  - solar-created value
  - bill totals
  - financing costs
- Export credit is not presented in a way that implies accidental double-counting
- Selected-period repayments are shown as part of investment/payoff treatment,
  not energy-bill treatment
- Tests cover at least one worked example demonstrating the relationship between
  self-consumed solar value, export credit, total solar value, bill reduction,
  and net solar position

## Dependencies

- `U-054`
- `P-049`
- Existing Range History and tariff-priced billing calculations
