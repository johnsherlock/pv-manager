# Q-014: Add worked-example validation coverage for Range History financials

## Objective

Protect the Range History financial interpretation model with worked examples
so metric definitions, wording, and calculations do not drift apart as the page
evolves.

## Scope

- Add automated coverage for the clarified financial model introduced by
  `FE-015`
- Create worked examples that exercise the relationship between:
  - self-consumed solar value
  - export credit
  - total solar value
  - import bill
  - import bill without solar
  - bill reduction from solar
  - repayments
  - net solar position
- Include at least one example that covers immersion diversion vs export
- Ensure tests make it hard to reintroduce misleading label/calculation
  combinations such as double-counting export credit

## Out of Scope

- End-to-end supplier bill reconciliation
- Manual QA-only notes without automated assertions

## Acceptance Criteria

- Automated tests cover the core Range History financial relationships defined
  in `U-054` and `U-055`
- At least one worked example protects the distinction between solar-created
  value, bill impact, and financing cost
- At least one worked example covers immersion trade-off interpretation
- The tests are readable enough to serve as living examples of the intended
  financial model

## Dependencies

- `P-054`
- `P-055`
- Existing range and billing test harnesses
