# U-034: Create the dedicated Historical Day page and content model

## Objective

Create a distinct Historical Day page for past dates using the shared
single-day components so users can review a completed day without the confusion
of realtime labels, realtime metrics, or future-looking content.

## Scope

- Add a new historical day page/route for past dates
- Define date navigation rules between `/live` and Historical Day
- Use neutral historical wording across shared modules
- Replace the lower live/future-looking section with same-day historical notes
- Add touch-friendly previous/next affordances that align with swipe behavior

## Out of Scope

- Rankings or percentile callouts
- Same-day-last-year comparisons
- Best-day insights
- Range history layout work

## Constraints

- Historical Day must be visually and functionally distinct from Live
- Desktop supports button and date-picker navigation
- Mobile and tablet support swipe left/right to move one day at a time
- Swipe direction should match chronology:
  - swipe left -> next day
  - swipe right -> previous day
- Swiping forward from yesterday to today returns to Live
- Swiping must be ignored where it would hijack chart interaction or normal
  scrolling

## Dependencies

- `P-029`
- `U-024`

## Acceptance Criteria

- Historical Day has its own route/page and uses shared single-day components
- Historical Day shows:
  - selected date header
  - trust/data-quality status
  - day totals
  - financial summary when tariff exists
  - day trend
  - solar coverage for the selected day
  - historical notes/story
- Historical Day hides:
  - realtime metric cards
  - right-now interpretation
  - next-few-hours / forecast / solar-context modules
- Historical copy uses neutral day wording rather than incorrect `today` copy
- Navigating to today returns the user to Live
- Swipe navigation works on touch devices and feels intentional rather than
  fragile

## Tests

- Route behavior between Live and Historical Day
- Historical copy assertions for labels and headings
- Rendering assertions for shown/hidden modules
- Touch swipe navigation behavior on mobile/tablet-sized viewports
- Guard behavior for swipe-disabled chart/interaction areas

## Notes

- Target copy changes include:
  - `Live trend` -> `Day trend`
  - `Today value` -> `Day value`
  - `Today so far` -> `Day totals`
  - `How much of the home solar has covered today` -> `Solar coverage for this day`
- This story defines the user-facing historical screen; comparative insights are
  intentionally left for follow-on work
