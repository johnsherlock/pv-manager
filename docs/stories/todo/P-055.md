# P-055: Build immersion trade-off metrics and comparison views

## Objective

Implement the first Range History immersion trade-off surfaces so users can see
how much surplus solar went to immersion, how much was exported, and what
export value was likely forgone by diverting that energy to hot water.

## Scope

- Implement the first immersion trade-off metrics defined by `U-055`
- Surface:
  - immersion diverted kWh
  - exported kWh
  - immersion vs export split
  - potential export credit foregone
- Add the chosen comparison treatment to Range History, such as a ratio view,
  chart, or supporting insight module
- Ensure the UI explains that this is a trade-off metric, not a generic loss or
  error state
- Reuse existing summary data where possible; add range-side aggregation and
  formatting where needed
- Add automated coverage for the metric relationships and selected-period
  aggregation behavior

## Out of Scope

- Appliance control automation
- Estimated hot-water replacement value unless `U-055` explicitly includes it
- Broader household optimization recommendations

## Acceptance Criteria

- Users can see how surplus solar split between immersion and export for the
  selected period
- Potential export value foregone by immersion diversion is visible and clearly
  labeled as a trade-off metric
- The implementation does not frame immersion diversion as inherently bad
- Tests cover the first-pass aggregation and value calculations

## Dependencies

- `U-055`
- `P-054`
