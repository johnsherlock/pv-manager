# P-042: Build Tariffs overview screen

## Objective

Implement the Tariffs overview/history screen in `apps/web`, giving signed-in
users a clear view of the active tariff, historical versions, reminder state,
the current weekly schedule shape, and the main actions into tariff editing.

## Scope

- Build the Tariffs overview route/screen within Settings
- Show current tariff summary
- Show compact schedule summary / preview for the active tariff version
- Show tariff version history / timeline
- Surface reminder / expiry state
- Link to create/edit tariff actions
- Handle empty, stale, setup-incomplete, and hard-error states

## Out of Scope

- No tariff create/edit form implementation
- No reminder email delivery
- No unrelated settings sections beyond the shell integration

## Dependencies

- `U-027`
- `P-041`
- `P-044`

## Design Reference

- [`docs/ui/settings.md`](../../ui/settings.md) — Tariffs section definition,
  Settings route structure (`/settings/tariffs`), and the completion-signal
  rule for the Tariffs card on the Settings home (produced by U-049)

## Acceptance Criteria

- Users can open the Tariffs section from Settings
- The active tariff is visually distinct from historical versions
- The active tariff includes a coherent summary of its weekly 30-minute schedule
- Reminder / expiry state is visible and understandable
- Empty and stale states preserve clear action paths into create/edit flows
- Desktop and mobile layouts remain coherent

## Notes

- This is the first full implemented settings section and should establish the
  quality bar for later Settings domains
