# P-042: Build Tariffs overview screen

## Objective

Implement the Tariffs overview/history screen in `apps/web`, giving signed-in
users a clear view of the active tariff, historical versions, reminder state,
the current weekly schedule shape, and the main actions into tariff editing.

This story is the production follow-up to `U-027`: it should supply the real
read path, page structure, and summary rendering for the overview screen, while
leaving actual tariff editing behavior to `P-043`.

## Scope

- Build the Tariffs overview route/screen within Settings
- Show current tariff summary
- Show compact schedule summary / preview for the active tariff version
- Show tariff version history / timeline
- Derive and render tariff schemes from stored weekly schedule data so the UI
  can group days that share the same 48-slot pattern
- Render one row per price period used by a scheme, including:
  - period label as read-only text
  - period rate as read-only text
- Surface reminder / expiry state
- Link to create/edit tariff actions
- Handle empty, stale, setup-incomplete, and hard-error states
- Ensure overview rendering remains coherent when:
  - a tariff version contains multiple schemes
  - a scheme contains multiple price periods
  - free-import periods exist

## Out of Scope

- No tariff create/edit form implementation
- No add/remove price-period controls on the overview
- No add/remove tariff-scheme controls on the overview
- No editor-time validation for overlapping or incomplete schedules
- No reminder email delivery
- No unrelated settings sections beyond the shell integration

## Dependencies

- `U-027`
- `P-041`
- `P-044`

## Design Reference

- [`docs/ui/settings.md`](../../ui/settings.md) — Tariffs section definition,
  Settings route structure (`/settings/tariffs`), and the completion-signal
  rule for the Tariffs card on the Settings home (produced by U-049)

## Acceptance Criteria

- Users can open the Tariffs section from Settings
- The active tariff is visually distinct from historical versions
- The active tariff includes a coherent summary of its weekly 30-minute schedule
- Weekly schedule data is grouped into schemes at render/load time in a way that
  matches the `U-027` model:
  - each day belongs to exactly one rendered scheme
  - schemes are grouped by identical 48-slot patterns
  - the periods shown in each scheme are the periods actually used by that
    scheme's slots
- Reminder / expiry state is visible and understandable
- Empty and stale states preserve clear action paths into create/edit flows
- Desktop and mobile layouts remain coherent
- The overview remains read-only:
  - no inline editing
  - no remove controls
  - no `Add price period` or `Add tariff scheme` actions
  - the main editing entry point is the existing `Edit` action for the active
    tariff (plus add-version actions where relevant)
  - period labels and rates read as display content, not editable inputs

## Notes

- This is the first full implemented settings section and should establish the
  quality bar for later Settings domains
- The overview should render valid stored data faithfully; correctness rules
  such as complete week coverage, no overlaps, and add/remove editing behavior
  belong to `P-043`
- Only the active tariff should render the full weekly schedule by default;
  historical versions can remain compact in the overview/history list
