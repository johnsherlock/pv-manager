# U-036: Mobile layout improvements for Historical Day and day-analysis charts

## Objective

Fix the mobile layout of the Historical Day screen so the chart is visible above the fold
on a 412×915 viewport, reducing wasted space in the control bar, page hero, chart header,
series legend, and chart height.

## Background

On a 412px-wide mobile viewport roughly 470px of chrome appears above the energy trend chart,
pushing it below the fold entirely. The main contributors are:

1. The control bar wraps to two rows (`flex-wrap`)
2. The page hero section has a description paragraph visible on mobile
3. The `DayTrendChart` card header includes a description paragraph visible on mobile
4. The series legend is a single-column grid on mobile (5 stacked rows ≈ 200px)
5. Chart heights are fixed at desktop sizes (`h-[400px]`, `h-[330px]`, `h-44`)

## Scope

- **Control bar** (`HistoricalDayScreen.tsx`): collapse to a single compact row on mobile.
  Show only `[<] [date] [>]` on the left and the trust badge on the right. Hide the clock
  and uptime badge on mobile (uptime detail is accessible via the trust badge tap).
- **Page hero** (`HistoricalDayScreen.tsx`): hide the description paragraph on mobile
  (`hidden sm:block`); keep the date heading.
- **Chart card header** (`DayAnalysis.tsx`): hide the description paragraph on mobile
  (`hidden sm:block`).
- **Series legend** (`DayAnalysis.tsx`): change from single-column on mobile to two-column
  (`grid-cols-2 lg:grid-cols-5`), reducing 5 rows to 3.
- **Chart heights** (`DayAnalysis.tsx`): make responsive —
  - Energy trend: `h-[260px] sm:h-[400px]`
  - Cost/value: `h-[220px] sm:h-[330px]`
  - Solar coverage: `h-32 sm:h-44`

## Out of Scope

- Any layout changes to the Live screen (shares `DayAnalysis` components but mobile
  layout concerns there are a separate story)
- New navigation patterns or information architecture changes
- Desktop layout changes

## Acceptance Criteria

- On a 412×915 mobile viewport the energy trend chart is visible above the fold without
  scrolling (or requires only minimal scroll)
- The control bar occupies a single row on mobile
- No content is clipped or overlapping on mobile at 412px width
- Desktop layout (≥1024px) is unchanged
- All existing functionality (series toggle, resolution switch, date navigation) works
  on mobile

## Notes

- Changes touch `HistoricalDayScreen.tsx` and `DayAnalysis.tsx` only
- The clock and uptime badge are secondary info on historical screens — hiding on mobile
  is acceptable; the trust badge already surfaces data-quality state
- Both screens (Live and Historical) use `DayAnalysis` — chart height and legend changes
  will improve the Live mobile layout as a side effect, which is acceptable
