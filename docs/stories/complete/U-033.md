# U-033: Build the first production Historical Day view

## Objective

Create the first dedicated Historical Day experience in `apps/web` so past dates
are shown as daily-review screens rather than as slightly modified versions of
the Live screen. The feature should keep Live focused on realtime state while
making day-to-day historical browsing fast and coherent on both desktop and
touch devices.

## Scope

- Add a dedicated historical day route/page for past dates
- Use neutral historical copy rather than live/today wording
- Support date navigation between Live and Historical Day
- Show historical:
  - day totals
  - day value
  - day trend
  - solar coverage
  - notes / interpretation
- Support swipe left/right navigation on mobile and tablet

## Out of Scope

- Rankings or percentile callouts
- Best-day insights
- Same-day-last-year comparisons
- Range history work
- New provider or persistence model changes

## Constraints

- Historical Day should be a distinct page experience, not a deeply conditional
  version of Live
- Live remains the home for realtime metrics, realtime interpretation, and
  future-looking modules
- Historical wording must not incorrectly imply that the selected day is still
  in progress unless the selected date is actually today
- Touch navigation must not break chart interaction or normal vertical scrolling

## Dependencies

- `P-029`
- `U-034`
- `P-028`
- `U-035`

## Acceptance Criteria

- Historical dates no longer render realtime-only modules
- A selected past day has its own route/page shell
- Historical copy does not incorrectly say `today`
- Navigating back to the current day returns the user to Live
- Swipe navigation works on touch devices without breaking chart interaction
- Desktop and touch navigation use the same date-to-date behavior

## Tests

- Verify historical routes render the historical experience instead of Live
- Verify navigating from a historical day to today returns to Live
- Verify touch swipe navigation works on mobile/tablet-sized viewports
- Verify chart interaction and vertical scroll still behave normally when swipe
  navigation is enabled

## Notes

- This is the umbrella feature story for the first production Historical Day
  slice
- `P-027` was already in use in the backlog, so the refactor story for this
  feature is tracked as `P-029`
- Comparative historical insights remain intentionally deferred to later stories
