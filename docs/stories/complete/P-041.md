# P-041: Build Settings shell and setup-completion home

## Objective

Create the signed-in Settings area, section navigation, and setup-completion
home so optional user-managed data has a clear product home after required
provider setup is complete.

## Scope

- Add the main Settings route / shell in `apps/web`
- Add Settings section navigation for desktop and mobile
- Implement the Settings home / overview state
- Surface setup-completion status for the main user-managed domains
- Add Tariffs as the first fully navigable settings section
- Add placeholders or secondary cards for later sections only where they help
  users understand what can be completed next

## Out of Scope

- No admin settings
- No full implementation of finance, solar, location, or notification settings
- No provider-setup gate logic from `FE-007`
- No public demo-mode settings behavior

## Dependencies

- `U-049`
- `U-015`

## Design Reference

- [`docs/ui/settings.md`](../../ui/settings.md) — Settings IA, shell layout,
  setup-completion card model, section list, route structure, and navigation
  bar integration rules (produced by U-049)

## Acceptance Criteria

- Signed-in users can open a dedicated Settings area
- Settings navigation works on desktop and mobile
- Users can understand which setup areas are complete vs optional and still
  available to add
- Tariffs is present as the first implemented settings section
- Unimplemented settings domains do not feel broken or misleading

## Notes

- This story is about structure and navigation first; Tariffs becomes the first
  real deep section through later stories

### Status: Done
