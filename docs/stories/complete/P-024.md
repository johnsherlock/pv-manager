# P-024: Set up the local Postgres development environment

## Objective

Make the rewrite app reliably runnable on this machine by defining and documenting
the local Postgres path required for schema commands, seed data, and the first
single-user app slice.

This story should remove the current ambiguity around `DATABASE_URL`, local DB
startup, and the steps needed to run `db:push`, `db:seed`, and app reads against
the rewrite schema.

## Scope

- Define the supported local Postgres setup for `apps/web`
- Add or update any local-dev helper files needed to make setup reproducible
- Document the exact steps required to:
  - start or connect to a local Postgres instance
  - create the local database
  - set `DATABASE_URL`
  - run `npm run db:push`
  - run `npm run db:seed`
- Verify that the seeded single-user dataset loads successfully

## Out of Scope

- No hosted database setup
- No production secrets or deployment environment wiring
- No multi-developer environment matrix
- No Supabase auth or remote platform setup

## Expected Implementation

- Add whichever of the following is the most maintainable fit for this repo:
  - `.env.example`
  - local setup notes
  - Docker Compose file
  - npm script helpers
- Verify the end-to-end local path in `apps/web`
- Keep the instructions explicit enough that another agent can follow them without guessing

## Constraints

- Prefer the simplest supported local path over a platform-heavy setup
- Keep everything scoped to the rewrite app in `apps/web`
- Avoid introducing unnecessary infrastructure if a normal local Postgres install is sufficient
- The final documented path must work with the existing Drizzle config and seed script

## Dependencies

- `P-018`
- `P-021`

## Acceptance Criteria

- A documented local Postgres path exists in-repo
- `DATABASE_URL` requirements are explicit
- `npm run db:push` works against the documented local DB setup
- `npm run db:seed` works against the documented local DB setup
- The seeded single-user data can be inspected or queried successfully afterward

## Tests / Verification

- Verify the documented steps from a clean local state as far as practical
- Confirm that Drizzle commands complete without manual guesswork
- Confirm that the seed script runs after schema sync

## Notes

- This story is now a direct prerequisite for the seeded single-user app flow
- Treat setup clarity as part of the deliverable, not an afterthought
