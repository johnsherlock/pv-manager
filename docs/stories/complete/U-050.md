# U-050: Define Finance setup and investment capture UX

## Objective

Define the signed-in Finance settings experience so users can record how their
solar installation was paid for, understand what information is required, and
unlock downstream payback and investment-recovery features through one simple
flexible finance profile rather than a branching mode-based form.

This story should turn the current Finance placeholder in Settings into a clear
product contract for both data capture and setup-completion status.

## Scope

- Define the `/settings/finance` page structure for desktop and mobile
- Define the completion rule for the Finance card on `/settings`
- Define the flexible finance input model:
  - investment date
  - upfront payment
  - monthly repayment amount including interest
  - repayment duration in months
- Define validation rules:
  - at least one of upfront payment or monthly repayment is required
  - if monthly repayment is entered, repayment duration is required
  - investment date is always required
- Define how the product derives:
  - total investment amount
  - remaining repayment months
  - outstanding scheduled balance
- Define empty, incomplete, editing, and saved-summary states
- Define the copy that explains what Finance unlocks in the product:
  - repayment coverage
  - recovered investment value
  - payoff estimate guidance

## Out of Scope

- Multiple phased investment events or expansion-history modeling
- Detailed downstream payback charts and projections beyond the UX contract
- Tax, depreciation, or grant-program modeling
- Provider outage or trust messaging beyond pointing to existing data-coverage states

## Acceptance Criteria

- Finance settings flow is documented clearly enough to implement without
  reopening the input model
- Required fields and validation rules are defined for the flexible finance profile
- Settings completion rules distinguish incomplete vs complete finance setup
- Saved-state copy explains the captured finance profile in user-facing terms
- Desktop and mobile behavior are documented
