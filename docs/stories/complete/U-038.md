# U-038: Design onboarding location capture and Live "Next" content model

## Objective

Define the user experience for capturing installation location during onboarding
and for presenting the lower `Next` section of the Live screen once weather and
sun-context data are available.

## Scope

- Define onboarding copy and interaction for:
  - exact address entry
  - eircode/postcode/zipcode entry
  - approximate area entry
- Define privacy messaging for exact versus approximate location modes
- Define missing-location prompts for the Live screen
- Define the 12-hour forecast rail content
- Define the 5-day forecast card content
- Define approximate-location labeling and confidence cues
- Define key screen behavior on desktop and mobile

## Out of Scope

- No implementation in `apps/web`
- No provider or schema decision work beyond what has already been documented
- No weather-alert or automation design
- No roof-relative solar modeling

## Expected Implementation

- Update UX documentation to specify:
  - onboarding entry points and helper copy
  - what the user sees when they skip location
  - what the user sees when location is approximate
  - what the user sees when API data is temporarily unavailable
  - after-sunset behavior for solar-context modules
  - the content hierarchy for the `Next` section

## Constraints

- The Live screen must remain realtime-first
- Weather and solar context must feel additive, not dominant
- Missing-location states should encourage setup without making the page feel broken
- Approximate location must be labeled clearly without sounding alarming

## Dependencies

- `U-031`
- `P-033`
- `P-034`

## Acceptance Criteria

- Onboarding copy covers address/eircode/postcode/zipcode and approximate-area entry
- Privacy messaging is defined clearly enough for implementation
- The 12-hour forecast rail content is specified
- The 5-day forecast card content is specified
- Desktop and mobile behavior is documented
- States are documented for:
  - no location
  - exact location
  - approximate location
  - API unavailable
  - after sunset

## Tests / Verification

- Review whether the `Next` section remains subordinate to Live telemetry
- Review whether approximate-location labeling is understandable and calm

## Notes

- This story should keep weather context tightly integrated with solar/daylight context
- The `Next` section should improve interpretation of upcoming solar opportunity,
  not become a generic weather dashboard
