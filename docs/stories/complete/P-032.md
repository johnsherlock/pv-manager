# P-032: Replace Recharts with Chart.js for day-analysis charts

## Objective

Swap the Recharts `<Area>` charts in `DayAnalysis` for `react-chartjs-2` equivalents so
chart transitions are canvas-rendered and match the smooth animation quality of the V1 app.

## Background

Recharts renders SVG and drives animation through React state updates. At 1440 data points
(minute resolution) this causes janky, stuttering transitions when navigating between
historical days. V1 uses Chart.js via `react-chartjs-2` which renders to a `<canvas>`
element and animates entirely off the React render cycle using `requestAnimationFrame`,
producing butter-smooth transitions at any data volume.

## Scope

- Replace the three `<AreaChart>` / `<Area>` charts inside `DayAnalysis.tsx`:
  1. Energy trend chart (generation, consumption, import, export, immersion series)
  2. Cost/value chart (import cost, savings, export credit series)
  3. Solar coverage chart
- Preserve all existing behaviour:
  - Per-series visibility toggling (`activeSeries`)
  - Resolution switching (1 min / 30 min / 1 hour)
  - Line vs cumulative view mode
  - Hover tooltips
  - Responsive sizing
- Remove the Recharts dependency once all charts are migrated

## Out of Scope

- Any chart used outside `DayAnalysis` (Live screen reuses the same components — if
  those also benefit, that is a side effect, not a goal)
- New chart types or new series
- Changes to data shape or loader logic

## Dependencies

- P-029 (DayAnalysis components already extracted and shared)

## Acceptance Criteria

- Energy trend, cost/value, and solar coverage charts render and animate smoothly on
  desktop and mobile at all three resolutions
- Navigating between historical days produces a smooth canvas transition with no jank
- Series toggling, resolution switching, and view mode all work as before
- Recharts is no longer listed as a dependency in `apps/web/package.json`

## Notes

- V1's `react-chartjs-2` config is available as reference for axis formatting, tooltip
  style, and animation settings
- Chart.js requires manual registration of components (`Chart.register(...)`) — keep this
  in a single shared setup file rather than per-component
- Canvas rendering means standard CSS hover states won't apply; use Chart.js tooltip and
  interaction APIs instead
