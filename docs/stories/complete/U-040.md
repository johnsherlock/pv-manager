# U-040: Range History content model and screen direction

## Objective

Define the Range History screen before implementation begins — what modules it
contains, how they are ordered, how the period selector works, how each state
variant renders, and where the design departs from or builds on the v1 historical
view. This story produces a documented direction spec, not code.

## Background

The v1 historical view (see screenshots in feature context) shows: a summary
header, an energy trend line chart, a per-day stacked bar chart, and a solar
coverage area chart. The rewrite adds financial charts and drops the redundant
third bar. The mid-fi layouts in `docs/ui/mid-fi-layouts.md` have a Range History
section that provides a starting skeleton; this story refines it into a buildable
direction.

## Scope

Produce or update `docs/ui/range-history.md` covering:

### Period selector
- Named presets: Last 7 days, Last 30 days, Last 3 months, Last 12 months,
  This month, This year
- Custom range as a secondary path (date picker), not the default
- Preset is the active context while data loads — do not reset on load

### Module hierarchy (desktop and mobile)

**Desktop:**
1. Period selector + trust / last-updated strip
2. KPI row: Savings €, Actual cost €, No-solar cost €, Export credit €, Avg
   solar coverage %
3. Tariff-change callout (when `health.hasTariffChange` is true) — near KPI row,
   not buried below charts
4. Energy trend line chart (import vs generation, toggleable series)
5. Per-day stacked bar chart (import total | generation breakdown: gen-consumed /
   exported / immersion)
6. Financial cost histogram (per-day actual net cost vs without-solar)
7. Period cost breakdown chart (donut: import cost / fixed charges / export credit
   / solar savings)
8. Solar coverage area chart (daily % of consumption from solar)
9. Export ratio chart (daily % of generated energy that was exported)
10. Payback progress bar (financed installations only — cumulative savings vs
    monthly finance payments)

**Mobile:** same order, full-width stacked, period selector collapses to a row of
pill presets with a "Custom" overflow button.

### Chart design notes
- Drop the redundant third bar from v1 (Overall Gen) — the generation breakdown
  stack already shows total generation
- Import bar is a single stack (total import) — rate-band stacking (day/peak/night)
  is added when P-039 band data is available in the range API
- Financial charts use banded billing when available; label as "simplified" when
  falling back to day rate only
- All charts use ECharts (consistent with the existing Live and Historical Day
  screens)

### State matrix
Document rendering rules for:
- **Loading**: skeleton placeholders per module, preserve selected preset
- **Empty / no data**: distinguish "no summaries for this range" from "range
  predates history"
- **Partial**: which days are missing; totals exclude them with a callout
- **Tariff-change**: callout near KPI row listing the version change date(s)
- **No tariff / savings-locked**: replace financial KPIs and charts with prompt
  cards; keep energy charts usable
- **Hard error**: single failure summary with retry

### Free import slot
Document the visual slot for free import (free Saturday hours or equivalent) in
the import bar and cost breakdown donut so it can be added later without a layout
redesign. Mark it as "not active in v1" rather than omitting entirely.

## Out of Scope

- No code
- No prototype tooling — this is a written direction doc, not a clickable prototype
- Free import classification logic (deferred to a follow-up story after
  `free_import_rule_json` is defined)

## Acceptance Criteria

- `docs/ui/range-history.md` exists and covers all sections above
- Module order and mobile/desktop treatment are unambiguous enough to implement
  from without reopening layout decisions
- State matrix covers all required variants
- Free import slot is documented as a future extension point
