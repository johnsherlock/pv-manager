# P-030: Add client-side historical day API route and cache module

## Objective

Create the infrastructure that allows `HistoricalDayScreen` to fetch and cache
historical day payloads client-side, so that adjacent-day navigation can be
served from memory rather than triggering a server render on every transition.

## Scope

- Add `GET /api/history/[date]` that returns the full historical day payload as
  JSON, extracting the fetch and computation logic already in `page.tsx`
- Add a client-side `dayCache` module (`src/live/dayCache.ts`) that stores
  fetched payloads in a module-level `Map<string, Promise<DayPayload>>` keyed
  by local date
- Expose `prefetch(date, today)` (guards against future dates, deduplicates
  in-flight requests) and `get(date)` helpers
- Export the payload type from the API route so both server and client share
  the same shape

## Out of Scope

- Changes to `HistoricalDayScreen` rendering or navigation behaviour (covered
  by P-031)
- Changes to the Live screen (separate concern)
- Persistent or disk-based cache

## Constraints

- The API route must produce the same output as the current `page.tsx` server
  render so P-031 can use either path without a data-shape change
- Prefetch must never request today or any future date
- The cache module must be pure TypeScript with no React dependencies so it can
  be tested without a DOM

## Dependencies

- `P-028`
- `U-035`

## Acceptance Criteria

- `GET /api/history/2026-03-30` returns a valid JSON payload matching the shape
  `HistoricalDayScreen` already receives from the server component
- Calling `prefetch('2026-03-30', today)` stores the result and does not issue
  a duplicate request if called again for the same date
- Calling `prefetch` with a date >= today is a no-op
- `get(date)` returns the cached payload synchronously once resolved, or `null`
  if not yet cached

## Tests

- API route returns 200 with valid shape for a past date
- API route redirects or returns 404 for today or a future date
- `dayCache.prefetch` deduplicates in-flight requests for the same date
- `dayCache.prefetch` is a no-op for today or future dates
- `dayCache.get` returns null before prefetch and the payload after

## Notes

- The API route is a thin wrapper — the business logic (normalizer, loader
  functions, financial estimate) already exists; this story just exposes it
  over HTTP
- The `Map` cache is module-level, meaning it persists across route navigations
  within a single browser session but is cleared on full page reload
