# P-031: Convert HistoricalDayScreen to client-side navigation with day cache

## Objective

Wire `HistoricalDayScreen` to the day cache introduced in P-030 so that
adjacent-day navigation is served instantly from the client rather than
triggering a full server render on every transition. The initial page load
(direct URL, bookmark, or calendar jump) continues to use the server-rendered
props path so first-load performance is preserved.

## Scope

- On mount, call `dayCache.prefetch` for the previous and next eligible
  adjacent days
- Intercept button and swipe navigation: if the target date is already cached,
  update component state directly and call `window.history.pushState()` to
  update the URL — no Next.js route navigation, no server round-trip
- If the target date is not yet cached (calendar jump to a random past date or
  a cold first swipe), fall back to `router.push()` as today, then prefetch
  again from the newly loaded date
- Remove the `startTransition` + shimmer bar approach since instant cache hits
  make it unnecessary for the common path; keep it only for the cache-miss
  fallback
- Handle the `popstate` browser event so the back/forward browser buttons
  remain consistent with the client-managed URL stack

## Out of Scope

- Changes to the Live screen
- Persistent cache (cache is session-scoped)
- View transitions / animation between days (separate story)

## Constraints

- The server-component path in `page.tsx` must remain intact — initial loads
  still go through SSR
- The URL must always reflect the currently displayed date so links and
  refreshes work correctly
- Cache misses must degrade gracefully to the existing server-render path
  without errors or blank screens

## Dependencies

- `P-030`

## Acceptance Criteria

- Navigating to a prefetched adjacent day renders instantly with no visible
  loading state
- The URL updates correctly on each client-side navigation
- Browser back/forward buttons navigate correctly after client-side transitions
- A cache miss (calendar jump, cold load) falls back to server render without
  error
- Prefetch is triggered after each navigation so the cache window slides
  forward

## Tests

- Cache-hit navigation updates state and URL without calling `router.push()`
- Cache-miss navigation falls back to `router.push()` and then prefetches
  adjacent days
- `popstate` handler navigates to the correct date from the history stack
- Prefetch is re-triggered after each successful navigation

## Notes

- The `window.history.pushState()` approach is the same pattern React Router
  and other SPA routers use internally — Next.js App Router does not intercept
  `pushState` calls, so the URL updates cleanly without triggering a server
  render
- This story is the core SPA-feel restoration for historical day browsing; once
  landed, back/forward swiping through recent days should feel identical to the
  V1 experience
