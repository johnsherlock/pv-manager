# U-051: Define multi-investment recovery and payoff insight surfaces

## Objective

Define the first finance insight surfaces unlocked by recorded system
additions so the app can explain:

- how much value the user's system has recovered so far
- whether selected-period savings are covering scheduled repayments
- what the rough payoff outlook is

The first pass should stay simple, whole-system focused, and honest about
incomplete history and approximation limits.

## Scope

- Define the first recovery/payoff insight set for signed-in users:
  - `Recovered so far`
  - `Repayment coverage for selected period`
  - `Approximate payoff outlook`
- Define `Recovered so far` using the app's existing net savings concept rather
  than inventing a new recovery metric
- Define the primary screen ownership for these insights, with Range History as
  the main home for the first pass
- Define whether any lighter summary treatment should appear elsewhere without
  making a separate home/landing experience part of this story
- Define whole-system-only behavior for the first pass; per-investment recovery
  attribution is explicitly not required here
- Define trust and wording rules when:
  - recorded history starts after one or more system-addition dates
  - tariff coverage is incomplete
  - provider coverage is incomplete
  - no system additions exist
- Define how repayment-coverage messaging behaves for:
  - upfront-only additions
  - repayment-backed additions
  - mixed portfolios containing both
- Define when to show:
  - recorded recovery from available stored history
  - estimated total recovery since investment date
  - no estimate at all because the estimate would be too weak or misleading
- Define copy that makes payoff outlook feel like guidance, not a promise

## Out of Scope

- Complex forecasting models
- Tax/accounting reporting
- Dedicated Data Health feature design beyond the finance-specific trust wording
- Detailed per-investment recovery attribution rules
- A separate lifetime-recovery screen or dashboard beyond the first chosen
  surface(s)

## Acceptance Criteria

- The first finance insight set is documented with clear screen ownership
- `Recovered so far` is defined explicitly as cumulative net savings from
  recorded history
- Selected-period repayment coverage rules are defined for month, year, and
  custom range views
- The first pass is explicitly whole-system focused and does not require
  per-investment recovery attribution
- Incomplete-history treatment clearly distinguishes recorded recovery from any
  estimated total recovery since investment date
- Payoff outlook rules are simple, explicit, and caveated appropriately
- Missing-investment, partial-history, and mixed-financing states have defined
  UX treatment
